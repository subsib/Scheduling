\section{Presentation}
\subsection{Recall of the subject}
The aim of the internship is to develop and integrate a new service for the real-time operating system HIPPEROS, 
that would provide the functionality of the FATFS library to users.
Currently, HIPPEROS uses a FATFS library directly, the aim is to introduce an indirection in order to 
avoid concurrent access that might occur.

\subsection{Current task}
The current task is still to enhance the implementation of the prototype 
already done of the file system service. 
Once the service is integrated as a library, it must be tested in order to 
evaluate the efficiency of the service, and before continuing the implementation 
with the write function.


Two weeks ago, the PR was almost finished on the C code. I was working 
on integration tests in Python in order to automate the process, and briefly, 
these two objectives are now reached.

\section{Flow description}
\subsection{First week 28/08 to 01/09}
The first week, I have been working on the end of the PR about the C code, 
and starting a new PR for the Python tests. My supervisor was not physically present so 
the others helped me to work during the whole week.

On Tuesday, I started to work on the library integration : The service had to migrate 
from prototype to library. This job has been done the friday
friday : service integration ok, compile ok.
End of review on the first request.

\subsection{Second week 04/09 to 08/09}
At the start of the second week, it was decided that the next task would be to test the
service in order to check the efficiency.
During the week, I have been working on that task and others around Jenkins. 
I just started to work also on the next task : the write function support.
I also add tests on a board, and not directly on virtual memory, that allows 
to have a better idea of the real performance pf the system.

\subsection{Next goal}

At this time, the last PR on the library integration has not been reviewed.
For the next week, the idea is to finish the benchmarks and start the work around the 
write function even if it has not yet been done.
